## 소켓
네트워크 상의 프로세스 간 통신을 위한 인터페이스  
OSI 7계층에서 응용 계층과 전송 계층 사이  
프로토콜, IP, Port로 정의 가능  
소켓은 서버와 클라이언트 모두 메시지 전송 가능한 양방향 통신  

<br>

## 서버
1. 소켓 생성
2. ip, port 바인딩
3. 운영체제의 listen() 메서드 호출
4. accept() 메서드로 클라이언트와 연결  
5. 데이터 송수신
6. 소켓 닫기

<br>

## 클라이언트  
1. 소켓 생성  
2. 통신을 하려는 서버의 ip, port 연결 시도  
3. accept() 메서드로 클라이언트 소켓 정보 반환  
4. 데이터 송수신  
5. 소켓 닫기
